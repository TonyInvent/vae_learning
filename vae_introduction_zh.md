# 变分自编码器：从第一性原理到机器人想象力

> 一份学习 VAE、世界模型，以及为什么 KL 散度不是一个神秘附加项的叙事指南。

---

## 写在前面

本文写给那些想以"理解思想"而非"记住公式"的方式来学习 VAE 的学生。我们将沿着一条长弧前进：

1. 一台机器人接收到太多传感器数据。
2. 普通自编码器压缩了数据，但压缩后的空间难以信任。
3. VAE 通过让编码变成概率分布来解决这个问题。
4. KL 散度成为存储信息的代价。
5. 同一个思想生长为世界模型、Dreamer 风格的智能体，以及从像素、演示和想象未来中学习的机器人策略。

目标不是记住 ELBO。目标是**感受**为什么 ELBO 必然出现。

---

## 1. 机器人淹没在像素中

想象桌面上有一只小型机器人手臂。一台相机俯视工作台。画面里有一个杯子、一个方块、一个夹爪、一些阴影，可能还有背景里的一根线缆。

机器人每执行一次动作，相机都会给它一张图像。一张小小的 64×64 RGB 图像就已经包含：

```text
64 × 64 × 3 = 12,288 个数字
```

但机器人真正关心的物理状态要小得多：

- 杯子在哪？
- 夹爪在哪？
- 夹爪是张开的还是闭合的？
- 物体离目标有多远？
- 是否发生了接触？

有用的状态也许只需要 10 个、20 个或 50 个数字来描述。而相机给了我们 12,288 个。

这是第一层张力：

```text
世界很简单。
观测很庞大。
机器人必须在自己完全理解图像之前就开始行动。
```

如果我们直接在像素上训练强化学习策略，策略必须同时解决两个问题：

1. 学习看。
2. 学习做。

这对稀疏奖励来说要求太高了。如果机器人在成功抓起杯子之后才能收到一次奖励，那么在此之前的上百万个像素几乎是沉默的。策略必须同时自己发现视觉结构、物体恒存性、几何关系和运动控制。

于是我们提出一个自然的问题：

**能不能先学一个小型的世界表征，然后让策略在表征上做决策？**

这就是自编码器进入故事的地方。

---

## 2. 第一次尝试：压缩，然后重建

自编码器是这个想法的最简版本。

```text
图像 x  →  编码器  →  隐代码 z  →  解码器  →  重建图像 x̂
```

编码器压缩图像。解码器尝试从压缩的代码中重建原始图像。如果重建效果好，我们希望代码包含了重要信息。

一个最简的 PyTorch 自编码器长这样：

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class Autoencoder(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, latent_dim),
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, input_dim),
            nn.Sigmoid(),
        )

    def forward(self, x):
        z = self.encoder(x)
        x_hat = self.decoder(z)
        return x_hat, z


def ae_loss(x_hat, x):
    return F.binary_cross_entropy(x_hat, x, reduction="sum")
```

这已经给了我们一些有用的东西。我们不用再把 12,288 个像素值喂给策略，而是可以喂一个 32 维或 64 维的隐向量。

但普通自编码器有一个隐藏的问题：**它学的是代码，而不是世界。**

让我们把这句话变具体。

假设两张相机画面几乎一模一样。两张画面中，杯子都在中间。唯一的区别是光照的微小变化。我们希望这两个隐代码靠得很近，因为机器人应该将这两个场景视为相似的。

但自编码器并没有被要求这样做。它唯一的任务是重建。它可以把相似的图像放到相距很远的位置——只要解码器知道如何从两处的代码分别映射回正确的图像就行。

隐空间可能会变成一簇私有地址：

```text
有效代码     有效代码                        有效代码
    *            *                               *

         解码器从未学过的空白区域

                          *       *
                      有效代码
```

如果你在这个空间里随便挑一个点，解码器可能会输出一堆毫无意义的东西。如果你在两个有效代码之间做插值，中间的路径可能穿过毫无意义的区域。如果一个机器人策略在这个隐空间中移动，微小的视觉变化可能导致表征的不稳定跳跃。

普通自编码器解决了压缩问题，但没有解决几何问题。

而对机器人来说，几何至关重要。

机器人想要的隐空间是：

- 相邻的场景有相邻的代码。
- 世界中微小的移动产生代码中微小的移动。
- 从代码空间中随机采样能解码出合理的观测。
- 动力学模型能根据今天的隐状态预测明天的隐状态。

简而言之，机器人需要的不仅是压缩空间，而是**可用空间**。

VAE 正是从这一需求中诞生的。

---

## 3. 核心转折：不要把图像编码成一个点

普通自编码器说：

```text
这张图像变成这个确定的隐点。
```

VAE 说的则更柔和、更有用：

```text
这张图像变成一小片可能的隐点云。
```

编码器不再输出单一向量 `z`，而是输出概率分布的参数：

```text
图像 x → 编码器 → 均值 μ(x), 方差 σ²(x)
```

然后我们采样：

```text
z ~ Normal(μ(x), σ²(x))
```

解码器再从这个采样的 `z` 重建图像。

```text
x → 编码器 → q(z|x) → 采样 z → 解码器 → x̂
```

记号 `q(z|x)` 的意思是：

```text
给定输入 x，编码器在隐代码 z 上的分布
```

这个小小的改变就是 VAE 的核心。

它为什么有用？

因为如果一幅图像由一个分布而非一个点来表征，模型就不能再依赖一个无限精确的地址。解码器必须学会从**邻近的样本**中重建图像。这种压力让邻域变得有意义。

如果编码器说：

```text
μ = 2.0
σ = 0.5
```

那么在训练中解码器会看到 1.7、2.1、2.4、1.9 这样的值。它必须学会：所有这些邻近的值大体上属于同一个底层场景。隐空间变得更平滑，因为解码器是在云上训练，而非在钉子上训练。

但仅此还不够。

编码器可以作弊——把每朵云都弄得很小：

```text
σ → 0
```

那么"分布"几乎又变成了一个点。我们又回到了披着概率外衣的普通自编码器。

所以 VAE 需要第二股力量。

那股力量就是 KL 散度。

---

## 4. VAE 内部的两股力量

VAE 被两种相互竞争的欲望拉扯着。

第一股欲望是重建：

```text
在 z 中保留足够多的信息以重建 x。
```

第二股欲望是正则化：

```text
让每个 q(z|x) 都靠近一个简单的共享先验 p(z)。
```

通常先验是标准正态分布：

```text
p(z) = Normal(0, I)
```

因此损失函数有两部分：

```text
VAE 损失 = 重建损失 + KL 惩罚
```

更明确地说：

```text
重建损失:
    解码器把输入重建得好不好？

KL 惩罚:
    编码器的分布相比 Normal(0, I) 有多昂贵？
```

这就是核心的谈判：

| 力量 | 它要求什么 | 如果它主导 |
|---|---|---|
| 重建 | 存储更多关于输入的细节 | 隐空间变得碎片化，退化为普通自编码器 |
| KL 散度 | 靠近共享先验 | 隐代码可能携带太少信息 |

一个好的 VAE 不是 KL 项"赢了"的 VAE。一个好的 VAE 是重建和 KL 达成有益妥协的 VAE。

隐代码只应在值得付费的时候才存储信息。

这是思考 VAE 中 KL 的最干净方式：

**KL 散度就是信息价格。**

编码器每把 `μ` 从零移开一次，就要付费。每把 `σ` 缩小到小于 1 以变得更确定，就要付费。如果那份额外的精度对重建有帮助，模型就付这个价格。如果没有帮助，模型就停止支付。

**VAE 学习的是一种信息经济。**

---

## 5. KL 散度：用它所回答的问题来解释

KL 散度经常以公式的形式被引入：

$$
D_{KL}(Q \| P) =
\mathbb{E}_{x \sim Q}
\left[
\log \frac{Q(x)}{P(x)}
\right]
$$

这个公式是对的，但它没有告诉你 VAE 为什么需要它。

让我们翻译一下。

比值：

$$
\frac{Q(x)}{P(x)}
$$

在问：

```text
在 x 这一点，Q 比 P 大概多少？
```

对数比值：

$$
\log \frac{Q(x)}{P(x)}
$$

把这个比较变成了代价。

在 `Q` 下的期望说：

```text
把这个代价在 Q 实际走到的地方取平均。
```

所以：

```text
D_KL(Q || P)
= 当样本实际上来自 Q 时，
  用 P 作为你的参考分布的平均代价。
```

在 VAE 中：

```text
Q = q(z|x)       编码器对某个输入给出的分布
P = p(z)         共享先验，通常是 Normal(0, I)
```

所以 KL 项在问：

```text
与默认的隐分布相比，
这个输入专属的隐分布有多昂贵？
```

如果编码器说：

```text
对于这张图像，z 应该在 μ = 5 附近，方差极小。
```

KL 就很大。编码器要求在远离默认区域的地方拥有一个特殊地址，而且精度要求很高。这很贵。

如果编码器说：

```text
对于这张图像，z 可以从接近 Normal(0, I) 的分布中采样。
```

KL 就很小。编码器没有对输入说出太多特定的信息。

**这就是为什么 KL 控制信息。** 为了存储关于 `x` 的信息，编码器必须让 `q(z|x)` 与通用先验不同。差异越大，付费越多。

### 一维图景

假设先验是：

```text
p(z) = Normal(0, 1)
```

现在想象编码器输出：

```text
q(z|x) = Normal(3, 0.1²)
```

这说的是：

```text
对于这张图像，z 必须非常接近 3。
```

这是一条强烈的信息。在先验下，`z = 3` 是可能的但不太常见。在 `q` 下，它极其常见。编码器创造了一个远离默认区域、尖锐且输入专属的分布。KL 收取大笔费用。

现在想象：

```text
q(z|x) = Normal(0.2, 0.9²)
```

这说的是：

```text
对于这张图像，z 只略微偏离默认。
```

它携带的信息更少，KL 收费也更少。

这就是高斯 KL 闭式解背后的直觉：

$$
D_{KL}
\left(
\mathcal{N}(\mu, \sigma^2)
\|
\mathcal{N}(0, 1)
\right)
=
\frac{1}{2}
\left(
\mu^2 + \sigma^2 - 1 - \log \sigma^2
\right)
$$

看看什么被惩罚了：

| 项 | 含义 |
|---|---|
| `μ²` | 把均值从零移开要付 KL 代价 |
| `σ²` | 让分布太宽也要付 KL 代价 |
| `-log σ²` | 让分布太窄同样要付 KL 代价 |

最后一项至关重要。它防止编码器把每个分布缩成一个极小的尖峰却不用付任何代价。

### KL 项到底对隐空间做了什么

KL 项不会魔法般地创造语义。它做的事情更谦虚，也更强大：

```text
它迫使所有输入专属的隐分布共享同一套邻域系统。
```

没有 KL，每个输入可以随意占据一个任意的私有地址。

有了 KL，每个输入专属的分布都必须与同一个先验保持合理地兼容。隐空间变得更紧凑、更平滑、更容易采样。

这就是为什么训练好的 VAE 可以生成新样本：

```python
z = torch.randn(16)       # 从 Normal(0, I) 采样
x_new = decoder(z)        # 解码出合理的样本
```

普通自编码器无法承诺这一点。它的解码器只能理解编码器产生的散落代码。VAE 的解码器则是在"有效代码应当生活在一个公共的、先验形状的区域"这种压力下训练的。

这是第一条深层联系：

```text
KL 正则化是将压缩转化为生成的东西。
```

## 6. ELBO：用概率语言讲的同一个故事

VAE 常常通过 ELBO —— 证据下界 —— 来引入。

模型假设存在一个隐藏的隐变量 `z`：

```text
z → x
```

从隐代码采样：

```text
z ~ p(z)
```

再从中生成数据：

```text
x ~ p_θ(x|z)
```

我们梦想最大化：

$$
\log p_\theta(x)
$$

意思是：

```text
让观测数据在模型下是可能发生的。
```

但计算 `p_θ(x)` 需要对所有可能的隐代码进行积分：

$$
p_\theta(x) =
\int p_\theta(x|z)p(z)dz
$$

这个积分通常很难算。

于是我们引入一个编码器：

$$
q_\phi(z|x)
$$

编码器是对以下问题的近似回答：

```text
如果我看到了 x，大概是哪个隐变量 z 生成了它？
```

ELBO 是：

$$
\mathcal{L}(x)
=
\mathbb{E}_{z \sim q_\phi(z|x)}
\left[
\log p_\theta(x|z)
\right]
-
D_{KL}
\left(
q_\phi(z|x)
\|
p(z)
\right)
$$

这就是同样的两股力量的故事：

```text
期望对数似然 = 重建得好
对先验的 KL  = 不要使用不必要的奇特编码
```

训练通常最小化负的 ELBO：

$$
\text{loss}
=
\text{重建损失}
+
\text{KL 损失}
$$

概率推导之所以重要，是因为它告诉我们 VAE 不仅仅是加了正则的自编码器。它是一个带有近似推断网络的生成模型。

但为了直觉，保留更简单的句子：

```text
VAE 花 KL 代价在 z 中存储信息，
当这些信息能帮助重建 x 时获得重建回报。
```

## 7. 重参数化技巧：把随机性挪开

还有一个技术障碍。

编码器输出 `μ` 和 `logvar`，然后我们采样：

```text
z ~ Normal(μ, σ²)
```

但梯度下降需要知道改变 `μ` 和 `σ` 会如何改变损失。原始随机采样操作似乎会切断这条路径。

重参数化技巧把采样重写为：

$$
z = \mu + \sigma \odot \epsilon,
\quad
\epsilon \sim \mathcal{N}(0, I)
$$

现在随机性存在于 `ε` 中，它与网络参数无关。网络通过普通的可微操作来控制 `μ` 和 `σ`。

在 PyTorch 中：

```python
def reparameterize(mu, logvar):
    std = torch.exp(0.5 * logvar)
    eps = torch.randn_like(std)
    return mu + std * eps
```

这个技巧很容易被低估。它是让神经网络能通过普通反向传播来学习概率隐变量模型的桥梁。

## 8. PyTorch 中一个完整的最小 VAE

这是一个用于 MNIST 等扁平图像的紧凑 VAE。

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class VAE(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20):
        super().__init__()

        self.fc1 = nn.Linear(input_dim, hidden_dim)
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)
        self.fc_logvar = nn.Linear(hidden_dim, latent_dim)

        self.fc2 = nn.Linear(latent_dim, hidden_dim)
        self.fc3 = nn.Linear(hidden_dim, input_dim)

    def encode(self, x):
        h = F.relu(self.fc1(x))
        mu = self.fc_mu(h)
        logvar = self.fc_logvar(h)
        return mu, logvar

    def reparameterize(self, mu, logvar):
        std = torch.exp(0.5 * logvar)
        eps = torch.randn_like(std)
        return mu + std * eps

    def decode(self, z):
        h = F.relu(self.fc2(z))
        return torch.sigmoid(self.fc3(h))

    def forward(self, x):
        mu, logvar = self.encode(x)
        z = self.reparameterize(mu, logvar)
        x_hat = self.decode(z)
        return x_hat, mu, logvar


def vae_loss(x_hat, x, mu, logvar):
    recon = F.binary_cross_entropy(x_hat, x, reduction="sum")
    kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + kl, recon, kl
```

VAE 的架构指纹是双头编码器：

```text
编码器 → μ
       → logvar
```

模型不输出一个代码，而是输出代码上的一个分布。

简单的训练循环：

```python
model = VAE().to(device)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(num_epochs):
    model.train()

    for x, _ in train_loader:
        x = x.to(device).view(x.size(0), -1)

        x_hat, mu, logvar = model(x)
        loss, recon, kl = vae_loss(x_hat, x, mu, logvar)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    print(
        f"epoch={epoch:03d} "
        f"loss={loss.item():.1f} "
        f"recon={recon.item():.1f} "
        f"kl={kl.item():.1f}"
    )
```

训练完成后，有三件重要的事值得尝试。

第一，重建：

```python
x_hat, _, _ = model(x)
```

第二，生成：

```python
z = torch.randn(64, latent_dim).to(device)
samples = model.decode(z)
```

第三，插值：

```python
mu_a, _ = model.encode(x_a)
mu_b, _ = model.encode(x_b)

alphas = torch.linspace(0, 1, steps=10).to(device)
z_path = torch.stack([(1 - a) * mu_a + a * mu_b for a in alphas])
images = model.decode(z_path)
```

如果 VAE 是健康的，插值应该感觉像是在连续空间中穿行，而不是在无关记忆之间跳跃。

## 9. 阅读训练信号

训练 VAE 时，不要只看总损失。分别观察重建项和 KL 项。

重建项告诉你：

```text
解码器利用 z 重建 x 的效果有多好？
```

KL 项告诉你：

```text
编码器为存储信息付了多少代价？
```

一些常见的模式：

| 模式 | 通常意味着什么 |
|---|---|
| 重建改善，KL 上升 | 模型正在学习使用隐代码 |
| 重建改善，KL 坍塌接近零 | 解码器可能在忽略 z |
| KL 增长极大 | 模型可能在退化为普通自编码器 |
| 早期 KL 很高而重建很差 | 正则化压力可能太早太强 |

最著名的失败模式是**后验坍塌**。

后验坍塌发生在：

```text
q(z|x) 变得几乎等于 p(z)
```

用人话说就是：

```text
编码器停止发送有用信息。
```

数学上就是：

```text
μ → 0
σ → 1
KL → 0
```

这可能在解码器足够强大、不需要听 `z` 就能建模数据时发生。语言 VAE 经常受到这个问题的困扰，因为自回归解码器可以从前面的词预测下一个词，从而忽略隐变量。

常见的修复方法：

| 修复方法 | 思路 |
|---|---|
| KL 退火 | 从较小的 KL 权重开始，逐渐增大 |
| Free bits | 不惩罚每个维度上少量的 KL |
| 更弱的解码器 | 让解码器更多地依赖于隐代码 |
| β < 1 | 当重建需要更多信息时降低 KL 压力 |

KL 退火通常是最容易尝试的第一个实验：

```python
kl_weight = min(1.0, global_step / warmup_steps)
loss = recon + kl_weight * kl
```

这让模型先发现有用的代码，然后逐步把这些代码塑造进更好的隐空间。

## 10. β-VAE：把信息价格调高

标准 VAE 的损失是：

$$
\text{loss}
=
\text{重建}
+
D_{KL}(q(z|x) \| p(z))
$$

β-VAE 加了一个系数：

$$
\text{loss}
=
\text{重建}
+
\beta D_{KL}(q(z|x) \| p(z))
$$

当 `β > 1` 时，信息变得更贵。模型被迫只存储最重要的变化因素。

这可以促进解耦。比如，在一个简单物体的数据集中，一个隐维度可能追踪旋转，另一个追踪大小，另一个追踪位置。

但取舍是真实的：

```text
更高的 β → 更干净、更压缩的因素
更高的 β → 更差的重建
```

对机器人学习来说，解耦很有吸引力，因为它可能产生可解释的变量：

```text
隐维度 3  = 物体 x 坐标
隐维度 7  = 夹爪开合程度
隐维度 11 = 相机角度
```

但机器人策略并不总是需要人类可读的因素。有时它只需要任何能最好地预测奖励和动力学的表征。所以 β-VAE 有用，但并非魔法。

更深刻的启示是：

```text
改变 β 就是改变信息的价格。
```

这句话把 β-VAE 连回了 VAE 的核心思想。

## 11. VQ-VAE：当隐空间用词汇而非坐标

通常的 VAE 使用连续隐变量。VQ-VAE 使用离散代码。

不同于：

```text
z = 一个实数向量
```

VQ-VAE 使用：

```text
z = 一个指向已学习代码本的索引
```

编码器产生一个向量，然后模型将它替换为最近的代码本条目：

```text
x → 编码器 → 连续向量
           → 最近的代码本向量
           → 解码器 → x̂
```

这就像把感知变成词汇。模型学习可复用的视觉标记：

```text
"这里的边缘"
"轮子状的纹理"
"空地板"
"物体角落"
"夹爪形状"
```

当未来是多模态的，离散隐变量就很有用。如果机器人走到走廊的分岔口，未来不是左和右的平均值，而是要么左边要么右边。一个类别代码比一个高斯团块更能自然地表达这种替代方案。

这也是离散隐变量在后续世界模型中——包括 DreamerV2 和 DreamerV3——变得重要的原因之一。

## 12. 为什么 VAE 对强化学习很重要

现在回到机器人。

VAE 给了我们四个有用的工具。

### 1. 状态压缩

不必：

```text
像素 → 策略 → 动作
```

我们可以用：

```text
像素 → VAE 编码器 → z → 策略 → 动作
```

策略看到的是一个更小、更平滑的状态。这可以提升样本效率，因为策略不需要仅凭奖励就能重新发现底层视觉特征。

### 2. 自监督学习

VAE 可以在没有奖励的观测上训练。

这很重要，因为机器人奖励很昂贵。一个机器人可能探索数小时也解决不了任务。但每一帧相机图像都能训练 VAE：

```text
观测 → 重建目标
```

机器人在学会行动之前先学会看。

### 3. 新颖性与好奇心

如果 VAE 对某个场景的重建效果很差，这个场景可能是陌生的。

这个重建误差可以成为一个探索信号：

```text
高重建误差 → 新颖状态 → 也许该多探索一下
```

这需要谨慎使用，因为噪声也会导致高误差。但基本直觉是强大的：一个学习到的世界模型可以告诉智能体它还不理解什么。

### 4. 隐空间想象

这是最大的一项。

如果我们能学到：

```text
z_t, 动作_t → z_{t+1}
```

那么智能体就可以在隐空间中推演可能的未来。

它不需要完美地预测每个像素。它只需要一个足够好的未来表征来做决策。

这个想法直接导向了世界模型。

---

## 13. 世界模型：机器人在隐空间中学会做梦

2018 年 David Ha 和 Juergen Schmidhuber 的世界模型论文让这个想法变得生动。

智能体被分成三个模块：

```text
V: 视觉
   帧 x_t → 隐状态 z_t

M: 记忆 / 动力学
   z_t, 动作_t, 隐藏状态 → 预测的下一个隐状态

C: 控制器
   z_t, 隐藏状态 → 动作
```

在最初的设置中：

- 视觉模型是一个卷积 VAE。
- 记忆模型是一个预测未来隐状态的 MDN-RNN。
- 控制器很小，有时只是一个线性策略。

比具体架构更重要的是其哲学转变：

```text
不要在像素空间中训练策略。
在模型对世界的压缩梦境中训练它。
```

VAE 让这个梦变得可见。既然 `z` 可以解码回图像，我们就可以看到模型在想象什么。这让这个想法异常具体：智能体不仅仅在优化隐藏的张量，而是在一个可解释的隐模拟器里学习。

为什么这里 VAE 很重要？

因为动力学模型需要一个预测可行的隐空间。如果相邻场景映射到无关的代码，学习 `z_t → z_{t+1}` 就会很困难。KL 塑造的隐空间给了动力学模型一个更平滑的目标。

同一条线索再次出现：

```text
KL 让表征更便宜、更平滑、更可预测。
可预测让想象成为可能。
想象让高效控制成为可能。
```

## 14. 从世界模型到 Dreamer

世界模型将视觉、记忆和控制分开。后来的方法将整个系统变得更加一体化。

PlaNet 引入了用于从像素进行规划的循环状态空间模型。Dreamer 随后完全从想象的隐推演中训练 Actor-Critic 策略。DreamerV2 使用离散隐变量并扩展到 Atari。DreamerV3 用更稳健的训练配方将同一家族的思想推进到多种领域。

核心对象是循环状态空间模型，通常称为 RSSM。

它维护两种状态：

| 状态 | 角色 |
|---|---|
| 确定性隐藏状态 `h_t` | 对过去的记忆 |
| 随机隐状态 `z_t` | 当前情境的不确定性与紧凑表征 |

在每一个时间步，模型有两个信念。

先验：

```text
p(z_t | h_t)
```

这是动力学模型在看到当前观测之前预测的东西。

后验：

```text
q(z_t | h_t, x_t)
```

这是编码器在看到当前观测之后推断的东西。

KL 项现在比较的是这两个：

$$
D_{KL}
\left(
q(z_t|h_t,x_t)
\|
p(z_t|h_t)
\right)
$$

注意这个转变。

在简单的 VAE 中，KL 问的是：

```text
编码器接近 Normal(0, I) 吗？
```

在循环世界模型中，KL 问的是：

```text
模型的预测与编码器事后看到的内容一致吗？
```

这是同一个思想的优雅复用。KL 仍然在对齐两个分布，但先验不再是一个固定的高斯，而是模型自己的预测。

**模型学会了让自己的想象与未来的感知保持一致。**

Dreamer 然后在这个想象中训练行为：

```text
当前隐状态
→ 想象未来的隐状态
→ 预测奖励和价值
→ 更新 Actor 和 Critic
```

对机器人来说，这非常有吸引力，因为真实动作既缓慢又危险。想象的推演是廉价的。你可以在与物理世界交互之间做很多内部学习更新。

## 15. 一个具体的机器人学示例：用于模仿学习的 CVAE

VAE 不仅用于图像。它们也可以建模动作。

考虑一个从人类演示中学习的机器人。一个人遥控机器人执行一个任务，比如打开一个容器或插入一个物体。数据集包含观测和专家动作。

一个简单的行为克隆策略学到的是：

```text
观测 → 动作
```

但操作常常是多模态的。从同一个观测出发，可能有多种有效策略：

- 从左边接近
- 从右边接近
- 抓住边缘
- 抓住侧面
- 慢慢移动，先对齐
- 快速移动，后续修正

如果策略对这些策略取平均，结果可能很糟糕。两个好动作的平均可能是一个没有专家会做的动作。

条件 VAE 提供了一个自然的解决方案。

训练时，编码器同时看到：

```text
观测
专家动作序列
```

并产生：

```text
q(z | 观测, 动作序列)
```

解码器接收：

```text
观测
采样的 z
```

并重建专家动作序列。

```text
观测 + 动作块 → 编码器 → z
观测 + z       → 解码器 → 重建的动作块
```

隐变量 `z` 可以表示演示的风格或模式：

```text
正在使用的是哪一种有效策略？
```

在推理时，机器人可以采样 `z` 或使用默认的隐变量值，生成一个连贯的动作序列。

这就是同一种 VAE 逻辑披上了新的外衣：

| 图像 VAE | 动作 CVAE |
|---|---|
| 重建图像 | 重建动作块 |
| 隐变量捕获视觉因素 | 隐变量捕获策略/风格 |
| KL 让图像隐变量可采样 | KL 让动作策略可采样 |

用于 ALOHA 系列低成本双臂操作工作的 Action Chunking with Transformers（ACT），正是用了这种 CVAE 思想来表示动作块。重要的概念联系在于，VAE 不仅仅在生成漂亮的图像，它在解决一个更深的问题：

```text
一个观测如何支持多种可能的未来
而不把它们平均成无意义的东西？
```

隐变量回答说：

```text
条件化于观测。
用 z 来选择模式。
解码出一个连贯的未来。
```

## 16. 关于核心逻辑，该记住什么

如果你只能记住一句话，记住这句：

```text
VAE 将压缩变成了一条协商的信息通道。
```

编码器想要描述输入。解码器想要足够的信息来重建它。KL 项对编码器发送输入专属信息进行收费。

这场协商创造了我们从一开始就想要的属性：

| 期望的属性 | 它从何而来 |
|---|---|
| 压缩 | 隐维度小于输入 |
| 平滑 | 解码器在隐云的样本上训练 |
| 可采样 | KL 让分布与共享先验保持兼容 |
| 生成 | 我们可以从先验采样再解码 |
| 有用的世界模型 | 平滑的隐空间让动力学更容易学 |
| 多模态动作 | 隐变量可以在连贯的未来中选择 |

KL 项不是一个装饰性的正则化项。它是那个问出以下问题的机制：

```text
这条信息值得存储吗？
```

这个问题就是 VAE 在生成模型之外更重要的原因。机器人、世界模型和模仿学习都面对同一个基本困境：

```text
未来是不确定的。
观测太庞大。
智能体必须通过一个紧凑的信念来行动。
```

VAE 给了我们学习这个信念的最干净的方法之一。

## 17. VAE 仍然挣扎的地方

VAE 很强大，但它们不是通用答案。

### 模糊的重建

经典 VAE 经常产生模糊的图像。用逐像素损失训练的高斯解码器倾向于平均可能的输出。如果有多种合理的清晰图像，平均值可能就是模糊的。

这是后来的生成模型——如扩散模型——在高质量图像生成上占据主导地位的原因之一。

### 后验坍塌

如前所述，强大的解码器可以忽略隐代码。然后 KL 项趋于零，隐变量几乎不携带信息。

这不是一个小型的实现 bug，而是一种真实的训练动力学。

### 长距离预测

世界模型可以想象未来，但想象的推演会漂移。小的预测误差会累积。这在接触密集的机器人领域中尤其困难，因为接触时机的微小误差可能改变一切。

### 表征不自动等于控制

一个美丽的隐空间不能保证一个好的策略。表征必须保留对行动重要的东西，而不仅仅是对重建重要的东西。

这就是为什么许多现代系统将感知、动力学、奖励预测和策略学习一起训练。

## 18. 一个建议的学习路径

如果你想将这些变为现实，按这个顺序来做。

### 第 1 步：在 MNIST 上训练一个小 VAE

使用本文中的代码。绘制：

- 原始图像
- 重建图像
- 随机采样
- 插值路径
- 每个隐维度的 KL 值

不要着急。观察模型失败然后改进，这就是学习的一部分。

### 第 2 步：改变 KL 权重

尝试：

```text
β = 0.1
β = 1.0
β = 4.0
```

观察取舍：

```text
较低的 β → 更好的重建，更杂乱的隐空间
较高的 β → 更差的重建，更强的压缩压力
```

这会让你对"KL 作为信息价格"有切身感受。

### 第 3 步：从 MNIST 转移到类机器人观测

使用简单的模拟图像：

- 一个在正方形中移动的圆点
- 桌子上的一块积木
- 一个夹爪和一个物体

训练 VAE 并检查隐维度是否追踪位置、接触或物体身份。

### 第 4 步：学习隐动力学

训练一个小模型：

```text
z_t, 动作_t → z_{t+1}
```

然后向前推演并解码想象出的帧。

这是世界模型的最小版本。

### 第 5 步：带着这个故事去读论文

在有了直觉之后再去读经典论文。它们会感觉远没有那么抽象，因为你知道每个组件在解决什么问题。

## 最后的线索

VAE 始于一个朴素的问题：图像太大了，而机器人需要一个更小的状态。

普通自编码器压缩了图像，但它的隐空间可能变成一组私有地址。VAE 把代码从一个点变成了一个分布。这让解码器学会学习邻域而不是孤立坐标。

但仅有概率还不够。编码器仍然偏好极小、精确的云——除非我们为此收费。KL 散度就是那个收费者。它问的是每个输入专属的分布离共享先验有多远。通过这样做，它把隐空间变成了一条协商的信息通道：存储对重建有帮助的东西，丢弃不值得付费的东西。

从那一次协商出发，许多后来的思想自然随之而来。从先验采样给了我们生成。预测未来隐状态给了我们世界模型。在想象的隐推演中训练策略给了我们 Dreamer。将观测和隐策略变量一起条件化解码器，给了我们能选择单一连贯未来而非平均多个未来的机器人模仿策略。

这就是 VAE 的故事：它首先不是一个公式，而是一种让压缩表征变得平滑、可采样、对行动有用的方法。
